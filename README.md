# MudHelper
MUD 游戏移动端自动化助手，支持多服务器连接、指令挂机、循环执行、关键字触发、VIP 付费体系。
## 项目介绍
MudHelper 是我在玩mud游戏时突然来兴趣开发的，一款面向 MUD 文字游戏的移动端自动化辅助工具，提供稳定 TCP 连接、自定义指令序列、循环组 / 专属组 / 关键字执行、实时日志、用户系统、VIP 付费等功能。适用于个人、小范围玩家使用，不破坏游戏平衡，仅做效率提升。现在还有大量内容未完成。
## 已实现功能
用户注册、登录、邮箱验证码
用户信息、VIP 信息查询
多 MUD 服务器管理
游戏账号配置（多服务器）
启动 / 停止自动化任务
实时日志推送（WebSocket）
订单创建、支付状态查询
任务指令：普通指令、循环指令、专属组、关键字匹配
## 接口概览
1. 用户认证
POST /auth/register — 注册
POST /auth/login — 登录
POST /auth/send-code — 发送验证码
GET /auth/me — 获取当前用户信息
2. MUD 服务器
GET /mud/servers — 获取服务器列表
3. 用户配置（游戏账号）
POST /user/config — 设置游戏账号信息
GET /user/config — 获取游戏账号信息
4. 任务控制
POST /task/start — 启动任务
POST /task/stop — 停止任务
WS /task/log — 实时日志
5. 订单 & 支付
POST /order/create — 创建订单
GET /order/list — 获取订单列表
GET /order/info — 获取订单详情
## 技术栈
客户端：uni-app (Android)
服务端：Node.js + Express
数据库：MySQL
通信：TCP / WebSocket / HTTP
鉴权：JWT
验证码：SMTP 邮箱
支付：第三方聚合支付

## app使用说明
### 🔹 一、快速使用流程
打开 APP → 邮箱获取验证码登录
选择 MUD 服务器 → 填写游戏账号、密码
编辑指令（普通 / 循环 / 专属组 / 关键字）
启动任务 → 实时查看日志
用完停止任务，关闭连接
### 🔹 二、指令类型说明（最核心）
1. 普通指令
直接执行一条命令，可设置间隔。格式：命令|间隔秒示例：
```
look|1
hp|1
```
2. 循环指令
重复执行一段指令 N 次。格式：[名称:次数] 指令列表 [/名称]示例（循环 3 次）：
```
[打怪:3]
kill wolf
hp
```
[/打怪]
3. 专属组
提前保存常用指令，用 @组名 直接调用。
在「专属组」页面编辑：名称、指令、循环次数
使用时直接写：
@组名
示例：
```
@回血
@打怪
```
4. 关键字匹配
只有返回内容包含指定关键字，才执行下一条。格式：命令|间隔|关键字示例：
```
kill wolf|1|命中
```
只有返回出现 “命中”，才会继续执行下一条。
## 🔹 三、完整示例
```
look|1
hp|1
@回血
[打怪:5]
kill wolf|1|命中
hp|1
[/打怪]
```
